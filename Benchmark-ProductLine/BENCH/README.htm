   README FILE.

This web page explains the contents of the BENCH directory. 

First the names of the benchmarks and their contents is described, then the file format used in those files is explained.

We developed a random graph generator that produces files in that format. If you would like to use it to generate more examples please send us an email. 

If you have any questions, comments, suggestions, etc please let us know.

My email address is rlopez@cs.utexas.edu

Benchmark files

The following are the benchmark files:

  • CCExample: Example of connected components.

  • CycleExample1: Example to test cycle checking.

  • CycleExample2: Example to test cycle checking.

  • CycleExample3: Example to test cycle checking.

  • CycleExample4: Example to test cycle checking.

  • MSTExample: Example to test Minimum Spanning Trees.

  • NumberExample: Example to test Vertex Numbering.

  • SSSPExample: Example to test Single Source Shortest Path.

  • STRONGLYCC: Example to test Strongly Connected Components.

File Format

The benchmark files are plain text files with the following format:

NumVertices                                       number of vertices
NumEdges                                           number of edges
Undirected/Directed                           directed=1,   undirected=0 
SelfLoops                                            self-loops=1,  no self-loops=0
Unique/Repeated                                unique = 1,  repeated = 0
e11 e12                                                edge 1,  e11 is first vertex e12 is second vertex
...                                                          ... more edges ...
w1                                                         weight for edge 1
w2                                                         weight for edge 2
...                                                           ... more weights ...

Each row ends with a "\n" character.
